# 📮 ポスくん
## 未返信を毎朝 Slack に届けてくれる Bot

---

### こんな朝のメッセージが届きます

```
📮 ポスくんより、朝のお届け物です

【Chatwork】2件
  • 営業チーム  ← 田中さん（06/09 14:32）
  • 開発MTG    ← 鈴木さん（06/09 09:15）

【Slack】3件
  • #marketing_all  ← 山田さん（06/09 15:01）
  • DM: 佐藤さん   ← 佐藤さん（06/09 11:20）
  • #general        ← 高橋さん（06/09 08:45）

【Gmail（⚑要返信）】1件
  • 件名：お見積りのご確認  ← 高橋商事（06/09 10:00）

返事、お待ちしてる方が 6件 いるみたいですよ〜 🌤️
```

チャンネル名・ルーム名はタップするとそのメッセージに直接ジャンプできます。
PCを起動していなくても、毎朝8時に自動で届きます。

---

### 必要なもの

| 必要なもの | 備考 |
|---|---|
| GitHub アカウント | 無料プランでOK |
| Chatwork アカウント | APIトークン（設定画面で取得） |
| Slack アカウント | ワークスペースへのApp追加権限が必要 |
| Gmail アカウント | Google Cloud Console でOAuth設定が必要（オプション） |
| Claude Code | セットアップに使用（無料トライアルあり） |

---

### セットアップ方法

**Claude Code がステップごとに何をすべきか案内してくれます。**

Slack App の作成・API トークン取得・OAuth 設定・GitHub Secrets 登録など、
ブラウザ操作や設定作業は自分で行う必要がありますが、
Claude Code が「次にこれをやってください」と順番に指示してくれます。

```
① このリポジトリを fork する（Private に設定すること）
   https://github.com/sttech345/unreplied-report

② Claude Code を起動して、リポジトリのフォルダを開く

③「セットアップして」と伝える
  → Claude Code が CLAUDE.md を読み込み、手順を案内してくれます
```

所要時間：約 30〜60 分
（Slack App 作成・Gmail OAuth 設定・GitHub Secrets 登録などを含む）

---

### カスタマイズ

**対応済みスタンプを押したメッセージは翌朝から除外されます。**
どの絵文字を「対応済み」とみなすかは `config.yml` で自由に変更できます。

```yaml
slack:
  exclude_reactions:
    - white_check_mark  # ✅
    - eyes              # 👀
    - ok_hand           # 👌
```

実行時刻（デフォルト：平日8:00）の変更も1行で対応可能です。

### よくある質問

**Q. Chatwork も Slack も両方必要？**
A. どちらか一方だけでも動きます。不要なサービスは設定しなければスキップされます。

**Q. 送信先は自分だけ？**
A. はい。自分の Slack DM にのみ届きます。チームへの通知はしません。

**Q. セキュリティは大丈夫？**
A. APIキーは GitHub Secrets に保存され、コードには一切残りません。
   リポジトリを非公開（Private）にしておけばさらに安心です。

**Q. Gmail には対応している？**
A. 対応しています。`⚑要返信` ラベルが付いたメールを収集して報告します。ラベルは claude.ai のルーティン機能で自動付与されます。

### 注意事項（既知の制限）

| 制限 | 影響 | 回避策 |
|---|---|---|
| Slack / Chatwork：月曜は72時間に拡大 | 金〜日の未返信を月曜朝に拾います（対応済み）。GW・年末など4日以上の連休は手動確認を推奨 | — |
| Slack：@here / @channel 未対応 | グループ向けメンションは検出されない | Slack のリマインダー機能で個別管理 |
| Chatwork グループ：`[To:ID]` 形式のみ対象 | 返信リプライ形式のメンションは検出されない | 送り手に `[To:ID]` 形式を依頼 |

詳細・将来の改善方針は `CLAUDE.md` の「既知の制限事項と懸念点」セクションを参照してください。
